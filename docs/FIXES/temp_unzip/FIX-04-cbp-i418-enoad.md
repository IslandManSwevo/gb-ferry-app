# FIX-04: US CBP Form I-418 & USCG eNOAD — Correct Data Model & Submission Logic

## Regulatory Background

When a Grand Bahama Ferry vessel arrives at or departs from a US port (e.g. Fort Lauderdale / Port Everglades), two overlapping federal requirements apply:

### CBP Form I-418 (Passenger List / Crew List)
- Prescribed under **8 CFR 251.1, 251.3, and 251.4** and **Sections 231 and 251 of the Immigration and Nationality Act (INA)**.
- The master or agent of every commercial vessel arriving from a foreign place must submit Form I-418 to CBP at the port of immigration inspection.
- **On departure**, the operator must update the I-418 with crew additions, separations, and trip departure details per **8 CFR 251.3**.
- **Electronic submission** is now required through CBP's ACE system (Vessel Examination and Compliance System — VECS), replacing the former paper-only process per the December 2021 IFR (effective February 28, 2022). Paper Form I-418 is now only used as a fallback when the USCG eNOAD system is down.
- Each vessel must have an **ACE Portal Vessel Agency account** for VECS submission.

### USCG eNOAD (Electronic Notice of Arrival/Departure)
- Required under **33 CFR 160.201–160.216** for commercial vessels over 300 GT arriving from a foreign port.
- Submitted through the USCG National Vessel Movement Center (NVMC) via the VECS system.
- Each voyage segment gets a unique **NOTICE ID#** generated by USCG — this ID must be tracked and used for any Update/Delete/Cancel submissions.
- **Do NOT reuse a previous segment's NOTICE ID** for a new initial eNOAD — this is a common submission error flagged in CBP's VECS Agent FAQs.

### Current Platform Gaps
1. The I-418 data model does not capture all required fields (visa type, alien registration number, longshore work declaration).
2. eNOAD segments are not tracked with their USCG-assigned NOTICE ID, making Update/Cancel impossible.
3. There is no departure reconciliation step (crew additions and separations since arrival must be submitted on departure per 8 CFR 251.3).
4. No ACE/VECS submission status tracking.

```prisma
// packages/database/prisma/schema.prisma

enum I418SubmissionStatus {
  DRAFT
  SUBMITTED_ACE       // Successfully submitted to CBP via ACE/VECS
  ACCEPTED            // CBP confirmed receipt
  REJECTED            // CBP rejected — see rejectionReason
  DEPARTURE_PENDING   // Arrival submitted; awaiting departure reconciliation
  DEPARTURE_SUBMITTED // Departure update submitted (crew changes, separations)
  FALLBACK_PAPER      // USCG system was down; paper form submitted
}

model I418Submission {
  id          String   @id @default(cuid())
  vesselId    String
  vessel      Vessel   @relation(fields: [vesselId], references: [id])

  // CBP-assigned receipt number after submission
  // Format: POE-YYMMDD-BadgeNumber-MilitaryTime
  cbpReceiptNumber  String?

  portOfEntry       String    // e.g. "Port Everglades, FL"
  arrivalDate       DateTime
  departureDate     DateTime?

  voyageNumber      String
  lastForeignPort   String    // Last port before US entry
  nextForeignPort   String?   // First foreign port after US departure

  status            I418SubmissionStatus @default(DRAFT)
  rejectionReason   String?

  // Longshore work declaration (required on I-418)
  // 8 CFR 251.1: "Whether longshore work is to be performed"
  longshoreWorkDeclared Boolean @default(false)
  longshoreWorkException String? // Which INA exception applies if longshore work

  // Crew entries for this submission
  crewEntries       I418CrewEntry[]

  // Linked eNOAD segment
  eNoadSegmentId    String?   @unique
  eNoadSegment      ENoadSegment? @relation(fields: [eNoadSegmentId], references: [id])

  // Audit
  submittedByUserId String?
  submittedAt       DateTime?
  lastUpdatedAt     DateTime  @updatedAt
  createdAt         DateTime  @default(now())

  @@map("i418_submissions")
}

model I418CrewEntry {
  id              String         @id @default(cuid())
  submissionId    String
  submission      I418Submission @relation(fields: [submissionId], references: [id])

  crewMemberId    String
  crewMember      CrewMember     @relation(fields: [crewMemberId], references: [id])

  // I-418 required fields per 8 CFR 251.1
  rank            String         // As stated on the I-418 form
  nationality     String         // ISO 3166-1 alpha-3 country code
  dateOfBirth     DateTime
  placeOfBirth    String

  // Travel document
  passportNumber  String         // Encrypted at rest (see FIX-05)
  passportCountry String
  passportExpiry  DateTime

  // US visa / immigration status (required for non-US crew)
  // 8 CFR 251.1: alien registration or visa details
  visaType        String?        // e.g. "C1/D", "B1/B2", "TN"
  visaNumber      String?        // Encrypted at rest
  visaExpiry      DateTime?
  alienRegistrationNumber String? // Encrypted at rest — for LPR crew

  // Change flags (for departure reconciliation)
  joinedAfterArrival   Boolean @default(false)  // Added to crew after US arrival
  separatedBeforeDeparture Boolean @default(false) // Discharged at US port
  separationPortCity   String? // City where crew member was discharged

  createdAt DateTime @default(now())
  updatedAt DateTime @updatedAt

  @@map("i418_crew_entries")
}

model ENoadSegment {
  id          String   @id @default(cuid())

  // USCG NVMC assigns a NOTICE ID# for each initial eNOAD segment.
  // This is NOT the same as a voyage number and must NOT be reused across segments.
  // Reference: CBP VECS Agent FAQs
  uscgNoticeId  String?  @unique   // Assigned after initial submission

  vesselId      String
  vessel        Vessel   @relation(fields: [vesselId], references: [id])

  // Segment details (33 CFR 160.206)
  segmentType   String   // "ARRIVAL" | "DEPARTURE" | "UPDATE" | "CANCEL"
  portOfCall    String
  eta           DateTime
  etd           DateTime?
  actualArrival DateTime?

  status        String   @default("PENDING")  // PENDING, SUBMITTED, ACCEPTED, REJECTED

  // Linked I-418 submission for this port call
  i418Submission I418Submission?

  createdAt DateTime @default(now())
  updatedAt DateTime @updatedAt

  @@map("enoad_segments")
}
```

### 2. I-418 Generation Service

```typescript
// apps/api/src/modules/cbp/i418.service.ts

import { Injectable, Logger } from '@nestjs/common';
import { PrismaService } from '../prisma/prisma.service';
import { EncryptionService } from '../security/encryption.service';
import { AuditService } from '../audit/audit.service';

@Injectable()
export class I418Service {
  private readonly logger = new Logger(I418Service.name);

  constructor(
    private readonly prisma: PrismaService,
    private readonly encryption: EncryptionService,
    private readonly audit: AuditService,
  ) {}

  /**
   * Build the I-418 crew list for ACE/VECS submission.
   *
   * Regulatory basis:
   *   - 8 CFR 251.1 (arrival submission requirements)
   *   - 8 CFR 251.3 (departure reconciliation requirements)
   *   - INA Sections 231 and 251
   */
  async buildI418Payload(submissionId: string): Promise<I418Payload> {
    const submission = await this.prisma.i418Submission.findUniqueOrThrow({
      where: { id: submissionId },
      include: {
        vessel: true,
        crewEntries: { include: { crewMember: true } },
      },
    });

    // Decrypt sensitive fields for submission
    const crewList = await Promise.all(
      submission.crewEntries.map(async (entry) => ({
        rank: entry.rank,
        lastName: entry.crewMember.lastName,
        firstName: entry.crewMember.firstName,
        nationality: entry.nationality,
        dateOfBirth: entry.dateOfBirth.toISOString().split('T')[0],
        placeOfBirth: entry.placeOfBirth,
        passportNumber: await this.encryption.decrypt(entry.passportNumber),
        passportCountry: entry.passportCountry,
        passportExpiry: entry.passportExpiry?.toISOString().split('T')[0],
        visaType: entry.visaType ?? null,
        visaNumber: entry.visaNumber
          ? await this.encryption.decrypt(entry.visaNumber)
          : null,
        visaExpiry: entry.visaExpiry?.toISOString().split('T')[0] ?? null,
        alienRegistrationNumber: entry.alienRegistrationNumber
          ? await this.encryption.decrypt(entry.alienRegistrationNumber)
          : null,
        // Departure reconciliation fields
        joinedAfterArrival: entry.joinedAfterArrival,
        separatedBeforeDeparture: entry.separatedBeforeDeparture,
        separationPortCity: entry.separationPortCity ?? null,
      })),
    );

    const payload: I418Payload = {
      vesselName: submission.vessel.name,
      vesselNationality: 'BSH', // Bahamas — ISO 3166-1 alpha-3
      imoNumber: submission.vessel.imoNumber,
      voyageNumber: submission.voyageNumber,
      portOfEntry: submission.portOfEntry,
      arrivalDate: submission.arrivalDate.toISOString().split('T')[0],
      lastForeignPort: submission.lastForeignPort,
      longshoreWorkDeclared: submission.longshoreWorkDeclared,
      longshoreWorkException: submission.longshoreWorkException,
      masterCertification:
        'I certify that CBP baggage declaration requirements have been made known to all incoming crew.',
      crewList,
    };

    await this.audit.log({
      action: 'I418_PAYLOAD_BUILT',
      entityType: 'I418Submission',
      entityId: submissionId,
      details: { crewCount: crewList.length, portOfEntry: submission.portOfEntry },
    });

    return payload;
  }

  /**
   * Build the departure reconciliation update.
   * Required under 8 CFR 251.3: submit updated crew list showing additions
   * and separations that occurred during the US port call.
   */
  async buildDepartureReconciliation(submissionId: string): Promise<DepartureReconciliation> {
    const submission = await this.prisma.i418Submission.findUniqueOrThrow({
      where: { id: submissionId },
      include: { crewEntries: { include: { crewMember: true } } },
    });

    const additions = submission.crewEntries.filter((e) => e.joinedAfterArrival);
    const separations = submission.crewEntries.filter((e) => e.separatedBeforeDeparture);

    return {
      cbpReceiptNumber: submission.cbpReceiptNumber ?? '',
      departureDate: submission.departureDate?.toISOString().split('T')[0] ?? '',
      nextForeignPort: submission.nextForeignPort ?? '',
      crewAdditions: additions.map((e) => ({
        rank: e.rank,
        lastName: e.crewMember.lastName,
        firstName: e.crewMember.firstName,
        joinedAt: submission.portOfEntry,
      })),
      crewSeparations: separations.map((e) => ({
        rank: e.rank,
        lastName: e.crewMember.lastName,
        firstName: e.crewMember.firstName,
        separatedAt: e.separationPortCity ?? submission.portOfEntry,
      })),
    };
  }

  /**
   * Validate that an I-418 submission has all required fields before
   * it can be submitted to ACE/VECS.
   */
  async validateI418Completeness(submissionId: string): Promise<{
    valid: boolean;
    errors: string[];
  }> {
    const submission = await this.prisma.i418Submission.findUniqueOrThrow({
      where: { id: submissionId },
      include: { crewEntries: true, vessel: true },
    });

    const errors: string[] = [];

    if (!submission.vessel.imoNumber) {
      errors.push('Vessel IMO number is required for I-418 submission.');
    }
    if (!submission.voyageNumber) errors.push('Voyage number is required.');
    if (!submission.portOfEntry) errors.push('Port of entry is required.');
    if (!submission.lastForeignPort) errors.push('Last foreign port is required.');
    if (submission.crewEntries.length === 0) {
      errors.push('At least one crew entry is required.');
    }

    for (const entry of submission.crewEntries) {
      const name = `${entry.rank}`;
      if (!entry.passportNumber) errors.push(`${name}: passport number required.`);
      if (!entry.passportExpiry) errors.push(`${name}: passport expiry required.`);
      if (!entry.nationality) errors.push(`${name}: nationality required.`);
      if (!entry.dateOfBirth) errors.push(`${name}: date of birth required.`);
      // Non-US crew must have visa or alien registration
      if (
        entry.nationality !== 'USA' &&
        !entry.visaType &&
        !entry.alienRegistrationNumber
      ) {
        errors.push(
          `${name} (${entry.nationality}): non-US crew must have visa type or alien registration number.`,
        );
      }
    }

    return { valid: errors.length === 0, errors };
  }
}

interface I418Payload {
  vesselName: string;
  vesselNationality: string;
  imoNumber: string;
  voyageNumber: string;
  portOfEntry: string;
  arrivalDate: string;
  lastForeignPort: string;
  longshoreWorkDeclared: boolean;
  longshoreWorkException: string | null;
  masterCertification: string;
  crewList: any[];
}

interface DepartureReconciliation {
  cbpReceiptNumber: string;
  departureDate: string;
  nextForeignPort: string;
  crewAdditions: any[];
  crewSeparations: any[];
}
```

### 3. eNOAD Segment Tracker

```typescript
// apps/api/src/modules/cbp/enoad-segment.service.ts

import { Injectable } from '@nestjs/common';
import { PrismaService } from '../prisma/prisma.service';

@Injectable()
export class ENoadSegmentService {
  constructor(private readonly prisma: PrismaService) {}

  /**
   * Create a new initial eNOAD segment.
   * CRITICAL: Do NOT pass a uscgNoticeId on initial creation.
   * The NOTICE ID# is assigned by USCG NVMC after submission.
   * Reusing a previous segment's ID is a known submission error (CBP VECS FAQs).
   */
  async createInitialSegment(data: {
    vesselId: string;
    portOfCall: string;
    eta: Date;
    etd: Date;
  }) {
    return this.prisma.eNoadSegment.create({
      data: {
        ...data,
        segmentType: 'ARRIVAL',
        status: 'PENDING',
        // uscgNoticeId intentionally null — assigned after submission
      },
    });
  }

  /**
   * Record the USCG-assigned NOTICE ID# after successful initial submission.
   * This ID must be used for all subsequent Update/Delete/Cancel requests
   * for this segment.
   */
  async recordUscgNoticeId(segmentId: string, uscgNoticeId: string) {
    return this.prisma.eNoadSegment.update({
      where: { id: segmentId },
      data: { uscgNoticeId, status: 'SUBMITTED' },
    });
  }

  /**
   * Build an Update request using the existing NOTICE ID#.
   * Required when ETA, crew complement, or other details change after initial submission.
   */
  async buildUpdateRequest(segmentId: string, changes: Partial<{
    eta: Date;
    etd: Date;
    portOfCall: string;
  }>) {
    const segment = await this.prisma.eNoadSegment.findUniqueOrThrow({
      where: { id: segmentId },
    });

    if (!segment.uscgNoticeId) {
      throw new Error(
        `Segment ${segmentId} has no USCG NOTICE ID. ` +
        `Cannot submit update until initial eNOAD is accepted by NVMC.`,
      );
    }

    return {
      requestType: 'UPDATE',
      noticeId: segment.uscgNoticeId, // Required on all Update/Delete/Cancel
      ...changes,
    };
  }
}
```

## Migration Notes

1. **Existing CBP module**: The `CbpModule` already has SOAP gateway stubs for eNOAD. Wire `ENoadSegmentService.createInitialSegment()` to the submission flow so every submission starts by creating a segment record.
2. **NOTICE ID tracking** is not optional. Without it, CBP VECS rejections cannot be diagnosed and segment updates are impossible.
3. **Visa fields in the crew data model**: Add `visaType`, `visaNumber`, and `visaExpiry` to the `CrewMember` or `CrewAssignment` model if they don't already exist — these are mandatory I-418 fields for non-US crew.
4. **Longshore work declaration** must be surfaced as a required field in the UI before an I-418 can be submitted — currently there is no UI field for this.
5. **Encrypt** `visaNumber` and `alienRegistrationNumber` consistently with the same field-level encryption applied to `passportNumber` (see FIX-05).
