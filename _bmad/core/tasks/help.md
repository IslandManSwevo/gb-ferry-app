# BMAD Method Help

Welcome to the BMAD (Business-Managed Agentic Development) help guide. Below is a summary of the available modules, workflows, and commands to help you navigate and execute the BMAD Method.

## Modules Overview

| Module | Purpose |
| :--- | :--- |
| **BMM** (Business Management Module) | Project analysis, planning, requirements (PRD), and software development lifecycle management. |
| **BMB** (Business Management Builder) | Tools for creating, editing, and validating BMAD agents, modules, and workflows. |
| **TEA** (Test Engineering Agent) | Advanced test architecture, learning testing fundamentals, and automation. |
| **CIS** (Creative Innovation & Strategy) | Innovation frameworks, design thinking, problem-solving, and storytelling. |
| **CORE** (Core BMAD Utilities) | Global tools like brainstorming, multi-agent discussions (Party Mode), and documentation indexing. |

## Common Commands & Workflows

### 🕵️ Analysis & Planning (BMM)
- `/bmad-brainstorming`: Facilitate interactive brainstorming sessions.
- `/bmad-bmm-create-product-brief`: Collaborative discovery to create a product brief.
- `/bmad-bmm-create-prd`: Create a Product Requirements Document (PRD).
- `/bmad-bmm-create-architecture`: Create architecture solution designs.
- `/bmad-bmm-create-epics-and-stories`: Break requirements into actionable epics and stories.

### 💻 Implementation (BMM)
- `/bmad-bmm-sprint-planning`: Generate sprint tracking from epics.
- `/bmad-bmm-create-story`: Prepare a dedicated story file for development.
- `/bmad-bmm-dev-story`: Execute story implementation.
- `/bmad-bmm-code-review`: Perform adversarial code review.
- `/bmad-bmm-qa-automate`: Generate E2E automated tests for features.

### 🚀 Quick Flow (BMM Anytime)
- `/bmad-bmm-quick-spec`: Fast-track creation of implementation-ready specs for small changes.
- `/bmad-bmm-quick-dev`: Quickly implement small changes based on a tech spec.
- `/bmad-bmm-document-project`: Document existing projects for AI context.

### 🤖 Builder Tools (BMB)
- `/bmad_bmb_create_agent`: Create a new BMAD agent.
- `/bmad_bmb_validate_module`: Check BMAD modules against best practices.
- `/bmad_bmb_create_workflow`: Create a new structured BMAD workflow.

### 🧪 Quality & Testing (TEA)
- `/bmad-tea-teach-me-testing`: Learn testing through structured sessions.
- `/bmad-tea-testarch-test-design`: Create risk-based test plans.
- `/bmad-tea-testarch-automate`: Expand test automation coverage.

### ⚡ Innovation & Strategy (CIS)
- `/bmad-cis-design-thinking`: Human-centered design methodology.
- `/bmad-cis-innovation-strategy`: Architect business model innovation.

### 🛠️ Utilities (CORE)
- `/bmad-help`: Show available workflow steps or answer BMAD Method questions.
- `/bmad-party-mode`: Orchestrate discussions between all BMAD agents.
- `/bmad-index-docs`: Create a lightweight index for documentation.

## How to use a command
To run a specific workflow, simply type the command followed by any required context (like a file path or project directory). For example:
`/bmad-bmm-document-project C:\path\to\your\project`

> [!TIP]
> If you are unsure which step to take next, use `/bmad-help` or ask a specific agent like **Mary (BMM Analyst)** or **John (BMM PM)** for guidance.
